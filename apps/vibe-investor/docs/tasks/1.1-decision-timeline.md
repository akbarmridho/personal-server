# 1.1 Forced Decision Timeline on Every WAIT

Status: DONE
Priority: Iteration 1
Effort: Small
Impact: High — every WAIT gets a deadline, kills infinite deferral

## Problem

The TA skill issues WAIT recommendations that live forever. No expiration, no tracking, no escalation. AADI got "wait for retest at 10,900" across 5+ sessions without resolution.

## Changes

### `.opencode-config/skills/technical-analysis/SKILL.md` — DECISION section

Add after the existing WAIT definition:

Every WAIT recommendation must include:

1. Upgrade trigger — concrete, falsifiable condition that changes WAIT to BUY/HOLD
2. Downgrade trigger — concrete condition that changes WAIT to REMOVED/downgrade
3. Decision horizon — max trading days or desk-checks before forced resolution (default: 5 trading days or 3 desk-checks, whichever first)
4. Expiry action — what happens if horizon passes without trigger (default: downgrade to WATCHING)

A WAIT without these four fields is incomplete.

### `prompts/vibe-investor/main.md` — desk-check defaults

Add to symbol review step:

For every symbol carrying a WAIT with a decision horizon, check whether the horizon has passed. If yes, execute the expiry action. Do not silently renew — if the setup is still valid, re-underwrite with fresh horizon and fresh trigger levels.

### Symbol plan frontmatter — new `active_recommendation` schema

This is the base schema that 1.2 and 1.3 extend.

```yaml
active_recommendation:
  action: WAIT
  issued: 2026-03-25
  horizon_expires: 2026-04-01
  upgrade_trigger: "Daily close above 10,900 with volume"
  downgrade_trigger: "Daily close below 10,200"
  expiry_action: re-underwrite or downgrade to WATCHING
```
