# 1.5 Opportunity Cost Tracking

Status: PLANNED
Priority: Iteration 1
Effort: Small
Impact: Medium — makes inaction cost visible, pressures the agent to confront missed moves
Depends on: 1.2 (uses wait age data from `active_recommendation`)

## Problem

The agent tracks risk of being wrong (invalidation, stop loss, heat) but never tracks cost of being late (missed moves on READY names) or cost of being empty (cash drag). Missed opportunities are invisible.

## Changes

### `.opencode-config/skills/portfolio-management/SKILL.md` — desk-check review section

Add review step for READY symbols that moved away from trigger zone:

For every READY symbol where current price is >5% above the last recommended entry zone, compute and state the missed move: "AADI was READY at 10,500 entry zone on {date}. Current price is 12,000. Missed move: +14.3%."

Opportunity cost is not a reason to chase. It is a reason to re-evaluate whether the entry framework is too restrictive, whether the setup evolved into a different family, or whether the name should be re-underwritten at current levels.

### New memory file: `memory/notes/opportunity-cost.md`

```markdown
# Opportunity Cost Ledger

Last updated: {TRADING_DAY}

| Symbol | READY since | Entry zone | Current | Missed | Wait age | Status |
|--------|-------------|------------|---------|--------|----------|--------|
```

Rules:

- Updated every desk-check
- Missed move >10% triggers mandatory re-evaluation of entry framework
- Wait age >5 desk-checks triggers staleness: re-underwrite or expire
- Cumulative missed opportunity reported alongside portfolio heat in every synthesis

### `prompts/vibe-investor/main.md` — desk-check defaults

Add `memory/notes/opportunity-cost.md` to evidence-backed memory updates list.

### Memory template

Add `memory-templates/notes/opportunity-cost.md` with the table structure.
