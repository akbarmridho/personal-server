# 1.4 Pilot Entry Pathway in Portfolio Management

Status: DONE
Priority: Iteration 1
Effort: Medium
Impact: High — creates a path to deploy capital when the full 20-gate cascade can't be satisfied
Depends on: 1.1 (uses decision horizon for pilot expiry lifecycle)

## Problem

To issue a BUY, the agent must pass ~20 independent gates simultaneously. In a weak market, this is mathematically near-impossible. The system needs a lighter-weight entry path for small positions.

## Changes

### `.opencode-config/skills/portfolio-management/SKILL.md` — new section after Entry Discipline

Add "Pilot Entry Pathway" section:

When to use:

- Symbol is READY with explicit thesis, invalidation, and evidence grade 1-3
- Full BUY gates not satisfied (trigger absent, confirmation mixed, regime defensive)
- Agent has issued WAIT for 2+ desk-checks on this symbol
- Thesis has not degraded since WAIT was issued

Pilot sizing:

- Max: 0.5% of portfolio equity
- Default: 0.25%
- Must fit within heat budget and pass liquidity check

Pilot gates (reduced set — all must be true):

1. Thesis quality at least MEDIUM with evidence grade 1-3
2. Invalidation explicit, stop defined
3. Regime is not CAPITAL_PRESERVATION (DEFENSIVE or better)
4. Liquidity acceptable
5. No active portfolio override blocks the name

Skipped gates (vs full BUY):

- Perfect 15m trigger not required (daily location sufficient)
- Full confirmation not required (partial/developing OK)
- Flow alignment not required (neutral or better OK)
- Optimal RR not required (acceptable RR OK)

Pilot lifecycle:

- Pilot is a probe, not a commitment — does not change trade classification
- Full BUY trigger fires → scale to planned size through normal gates
- Setup fails or invalidation hit → exit pilot (loss capped at 0.25-0.5%)
- 3+ desk-checks with no trigger and no invalidation → exit pilot, downgrade to WATCHING
- Log with `entry_type: pilot` and the reduced gate set used

Pilot constraints:

- Max 2 active pilots at any time
- Don't count toward 50:30:10 (too small)
- Do count toward portfolio heat
- Cannot scale up without passing full BUY gates
- CAPITAL_PRESERVATION blocks all pilots
