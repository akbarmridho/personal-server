# 2.0 Shadow Scoring Bridge

Status: PLANNED
Priority: Iteration 2 (first task)
Effort: Medium
Impact: High — validates conviction scoring alongside existing outputs before replacing them
Depends on: Iteration 1 complete

## Problem

Switching from categorical verdicts to conviction scores in one shot is risky. The radical-redesign-proposal explicitly recommends a phased migration: add a scoring layer alongside existing outputs first, validate that it produces better decisions, then replace.

## What This Task Does

Add a scoring interpretation layer in the parent synthesis that converts existing skill outputs to 0-100 scores. Skills continue producing their current categorical outputs. The parent runs both the old gate cascade and the new scoring system in parallel, and logs both results.

This is Phase 2 from the radical-redesign-proposal migration path.

## Changes

### `prompts/vibe-investor/main.md` — synthesis section

Add a shadow scoring block to desk-check synthesis:

After the existing gate-cascade decision, also compute:

- Convert TA verdict + evidence to a 0-100 technical score using the rubric from 2.1
- Convert flow verdict + trust + conviction_pct to a 0-100 flow score
- Convert narrative strength to a 0-100 narrative score
- Convert fundamental quality + valuation to a 0-100 fundamental score
- Compute composite: `0.25×TA + 0.15×flow + 0.25×narrative + 0.20×fundamental + 0.15×portfolio_fit`
- Map composite to action tier (NO TRADE / WATCHLIST / PILOT / STARTER / STANDARD / HIGH CONVICTION)
- Log both the gate-cascade result and the scoring result in the desk-check artifact

The gate cascade still owns the actual decision. The scoring result is logged for comparison only.

### Desk-check artifact

Add a `shadow_scoring` section to desk-check output:

```yaml
shadow_scoring:
  technical_score: 62
  flow_score: 45
  narrative_score: 78
  fundamental_score: null  # not loaded this run
  portfolio_fit_score: 50
  composite: 64
  shadow_action: STARTER
  actual_action: WAIT  # from gate cascade
  divergence: true  # shadow and actual disagree
```

## Validation Criteria

Run shadow scoring for 5-10 desk-checks. If the shadow scoring consistently produces more actionable results on names that later moved favorably (or correctly blocks names that later failed), proceed to 2.1-2.5 to replace skill outputs.
